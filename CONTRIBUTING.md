# Contributing to ARIA

Thank you for your interest in ARIA (Autonomous Risk Intelligence Agent). As an institutional-grade project, we maintain high standards for code quality and documentation.

## How to Contribute

1. **Fork the Repository**: Create your own fork of the ARIA codebase.
2. **Create a Branch**: Use professional naming conventions (e.g., `feat/integration-standard`, `fix/parsing-anomaly`).
3. **Draft a Pull Request**: Provide a detailed summary of your changes and why they are necessary.
4. **Pass CI/CD**: Ensure all GitHub Action builds are passing before requesting a review.

## Standards

- **Code Style**: Follow PEP 8 for Python and standard ESLint rules for Next.js.
- **Commit Messages**: Use Conventional Commits (e.g., `feat:`, `fix:`, `docs:`, `chore:`).
- **Security**: Never commit API keys or forensic documentation datasets.
